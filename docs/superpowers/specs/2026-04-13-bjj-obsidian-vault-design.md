# BJJ Analysis - Obsidian Vault & ML Learning Project

## Context

Greig has an existing BJJ video analysis prototype: a Python script that uses Claude Vision API to identify positions from rolling footage and generate HTML coaching reports, plus React components for visualising the position taxonomy and transition graph. The project needs to be structured as an Obsidian vault, pushed to GitHub, and expanded with an ML learning path so Greig can eventually train a custom position classifier. Greig is new to both Obsidian and ML, with BJJ recordings from mixed sources (phone, GoPro, etc).

## Design

### Repository = Obsidian Vault

The repo root IS the vault. Opening the folder in Obsidian gives a fully functional knowledge base. Code lives in `tools/` which Obsidian ignores.

### Position Notes

Auto-generated from `taxonomy.json` by `generate_vault_notes.py`. Each note has:

- YAML frontmatter: `category`, `dominance`, `tags`
- Techniques section with `[[wikilinks]]` to Technique notes
- Transitions section with `[[wikilinks]]` to connected positions (from `valid_transitions` in taxonomy)
- Blank "My Notes" section for personal observations

### Technique Notes

Auto-generated from the JSX taxonomy data (which has richer technique lists). Each note has:

- YAML frontmatter: `tags`, `positions` (which positions use this technique)
- Description placeholder
- Links back to positions where the technique applies
- Blank "Tips" and "Drill Ideas" sections

### Roll Analysis Notes

Generated by `analyse_roll.py --markdown` (new output mode). Each note has:

- YAML frontmatter with date, partner, duration, scores (queryable by Dataview)
- Summary, key moments with `[[wikilinks]]` to positions/techniques
- Top improvements and strengths

### ML Learning Path Notes

7 self-contained lessons, each structured as:

1. **Goal** - what you'll understand/build
2. **Concepts** - explanation with BJJ analogies
3. **Hands-on** - practical exercise with your data
4. **Resources** - links to further reading
5. **Journal** - blank section for notes

Lessons:
- **00 Overview**: What we're building (position classifier), why (cheaper/faster/offline), the journey map
- **01 Image Classification Basics**: What is a classifier, supervised learning, training/test data, labels, accuracy. Exercise: manually label 10 frames
- **02 Data Collection Strategy**: Building a dataset from rolls, using analyse_roll.py for initial labels, data quality, how many frames per position
- **03 Labelling with Claude Vision**: Using the pipeline as a labelling tool, reviewing/correcting labels, building structured datasets
- **04 Training a Position Classifier**: scikit-learn Random Forest on resized images, train/test split, accuracy, confusion matrix
- **05 Pose Estimation with MediaPipe**: BlazePose skeleton extraction, 33 landmarks as features, retrain and compare
- **06 Evaluation and Iteration**: What worked, strategies for improvement, next steps (real-time, CNNs, session tracking)

### Obsidian Configuration

- `.obsidian/app.json`: set `tools/` and `assets/` as excluded from search
- Templates plugin enabled, pointed at `Templates/`
- Graph view will naturally show the position network via wikilinks
